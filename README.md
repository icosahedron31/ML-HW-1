#House Prices-Advanced Regression Techniques

# პირობა 
გვქონდა სახლების მონაცემები (79 ცვლადი) და ამ სახლების გაყიდვის ფასები. მოდელს უნდა ევარაუდა ფასები აქამდე უნახავ მონაცემებზე და ის შეფასდებოდა შემდეგი მეტრიკით: RMSE ჩვენს მიერ ნავარაუდები ფასის ლოგარითმებსა და ნამდვილი ფასების ლოგარითმებს შორის. 

# დატის გაყოფა 
დატის ზომა პატარა იყო მხოლოდ 1460 ჩანაწერი. გამოვიყენე 2 სტრატეგია: ჩვეულებრივი კროს ვალიდაცია 20%-იანი validation set-ით და KFold(K=5, K=10).  
როგორც მოსალოდნელი იყო KFold-მა აჯობა, თითოეული მათგანისთვის შედეგებს ქვემოთ განვიხილავთ. 

# პრეპროცესინგი 
## კატეგორიული ცვლადების რიცხვითებად გადაქცევა-
ის კატეგორიები რომელშიც 3-ზე ნაკლები ან ტოლი განსხვავებული მნიშვნელობა იყო (NaN ჩათვლით) გადავიყვანე one-hot-encoding-ით. 
სხვა ცვლადებისთვის: 
### პირველი სტრატეგია: 
უბრალოდ საშუალო სახლის ფასი ავიღე training set-დან 
ამ მეთოდმა საკმაოდ ნორმალური მოდელი მოგვცა KFold validation-ით და rfe-ით, 0.024 mse validation-ზე საშუალოდ(rmse=0.16) და train-ის საშუალო 0.016 mse (rmse=0.13). თუმცა ამ სტრატეგიას აქვს თავისი ნაკლები, მაგალითად შესაძლო target leak, თუმცა ამ ეფექტს შეამცირებს ის რომ საბოლოდ მოდელს ფასების ლოგარითმზე ვატრენინგებთ
### მეორე სტრატეგია: საშუალო ფასების smoothing
იმისთვის რომ data leak თავიდან აგვეცილებინა, გავაკეთე smoothing პირდაპირ საშუალოს მაგივრად ვხმარობდი შემდეგ ფორმულას: (n * mean + alpha * global_mean) / (n + alpha) სადაც alpha რაღაც კონსტანტა იყო n მოცემული კატეგორიული მნიშვნელობისთვის sample-ების რაოდენობა. შემდგომში alpha=20 და alpha=60 ვიყენებდი, თუმცა საბოლოოდ alpha=20-მა აჯობა. ეს მეთოდი დაგვაზღვევს იმისგან, რომ იშვიათი კატეგორიებისთვის target არ გაიleak-ოს
## NaN-ების შევსება 
აქ ვხმარობდით მარტივ სტრატეგიას: NaN-ებს ვავსებდით მედიანური მნიშვნელობებით. 
## შეჯამება
ცხადია ამ მონაცემების შევსებისას მხოლოდ train set-ს ვუყურებთ. იმ შემთხვევაში თუ ისეთი კატეგორია შეგვხვდება test-ში ან ვალიდაციაში რომელიც train-ში არ გვხვდებოდა, მას რიცხვითი სვეტებისთვის ისევ მედიანებით ვავსებთ, one-hot-encoding-ში კი უბრალოდ ყველა train-ში არსებული კატეგორიაში 0 გვაქვს. 

# Feature Engineering 
შეგვიძლია შემოვიტანოთ ახალი feature- სახლის სიძველე როდესაც ის გაიყიდა, რომლის მიღებაც ცხრილში არსებული ორი სვეტის სხვაობებით არის შესაძლებელი. ექსპერიმენტების ჩატარებისას აშკარა გახდა, რომ bottleneck არსებული feature-ები იყო, ამიტომ ცხრილში არსებული თითოეული რიცხვითი მაჩვენებლისთვის, რომელიც თავდაპირველად იყო მოცემული (ან თავდაპირველად მოცემული სვეტების NaN-ების შევსების შედეგად მივიღეთ) ჩავინიშნე კიდევ 2 დამატებითი feature: feature_log, feature_square რაც ზოგიერთი feature-ზე არაწრფივი დამოკიდებულებების დასადგენად დაგვეხმარებოდა.

# KFold

მეტრიკებად ძირითადად ვაბეჭდინებდი fold-ებს შორის train-mse-mean, val-mse-mean და val-mse-std. ეს საშუალებას იძლევა ვიმსჯელოთ რამდენად უკეთესია მოდელი ტრენინგზე ვიდრე ვალიდაციაზე და რამდენად დიდია ვალიდაციის შედეგის ვარიაცია fold-ებს შორის. დიდი ვარიაცია=overfit. ინტერპრეტირებადობისთვის ასევე ვითვლიდი RMSE-ს ნამდვილ ფასებსა და dapredict-ებულ ფასებს შორის(დოლარში და არა ლოგარითმში). feature-ების რეგრესიაზე გაშვებამდე

## შენიშვნა 

ყველა fold-ში ვალიდაციისას მიღებულ სახლის ფასებს თუ დავაკვირდებოდით, ერთ-ერთ fold-ში სხვაობა რეალურსა და ნავარაუდებს შორის გაცილებით მაღალი იყო (დაახლოებით. 100-140K), მაშინ როდესაც სხვა fold-ებში 20K-30K-იყო, იყო ერთი data point, რომელსაც მოდელი 1.4M ვარაუდობდა და სინამდვილეში 200K-დ გაიყიდა. ეს მიგვანიშნებს outlier-ების არსებობაზე დატაში. RFE-ს შედეგად მიღებული ფასებიდან ამოვიღე 20 ყველაზე დიდი ცდომილება და მათზე ანალიზი გავაკეთე, იმის გასარკვევად ამომეღო თუ არა დატიდან. 
### outlier ანალიზი
იქ სადაც განსაკუთრებით (500K+) დიდი ცდომილებები ფიქსირდებოდა არ იყო heavy-tail რაიმე ძვირად გაყიდული ან ძალიან იაფად გაყიდული სახლი, (-inf; Q1- 1.5IQR) + (Q3 + 1.5IQR; inf)-ში არ გვხვდებოდა. (Q1 -25 percentile, Q3-75 percentile, IQR- middle 50% spread)
feature space-ში დავითვალე რამდენად ახლოს იყო ეს point-ები ერთმანეთთან. ასევე დავითვალე 20 სიმულაციის რენდომად დაგენერირებული 20 data point რამდენად ახლოს იყო ერთმანეთთან. მივიღეთ რომ ეს point-ები გაცილებით გაფანტული იყო, ეს მიგვანიშნებს რომ ამხელა ცდომილებები არ გამოუწვევია რაიმე კლასტერს feature-space-ში და უბრალოდ დატიდან მიღებული ხმაურია. 
რადგან ეს მონაცემები გაყიდვის ფასით outlier-ებში არ შედიოდნენ გადავწყვიტე არ ამომეღო დატიდან და ბუნებრივ ხმაურად ჩამეთვალა.

# Smoothing
