#House Prices-Advanced Regression Techniques

#პირობა 
გვქონდა სახლების მონაცემები (79 ცვლადი) და ამ სახლების გაყიდვის ფასები. მოდელს უნდა ევარაუდა ფასები აქამდე უნახავ მონაცემებზე და ის შეფასდებოდა შემდეგი მეტრიკით: RMSE ჩვენს მიერ ნავარაუდები ფასის ლოგარითმებსა და ნამდვილი ფასების ლოგარითმებს შორის. 

#დატის გაყოფა 
დატის ზომა პატარა იყო მხოლოდ 1460 ჩანაწერი. გამოვიყენე 2 სტრატეგია: ჩვეულებრივი კროს ვალიდაცია 20%-იანი validation set-ით და KFold(K=5, K=10).  
როგორც მოსალოდნელი იყო KFold-მა აჯობა, თითოეული მათგანისთვის შედეგებს ქვემოთ განვიხილავთ. 

#პრეპროცესინგი 
#კატეგორიული ცვლადების რიცხვითებად გადაქცევა-
ის კატეგორიები რომელშიც 3-ზე ნაკლები ან ტოლი განსხვავებული მნიშვნელობა იყო (NaN ჩათვლით) გადავიყვანე one-hot-encoding-ით. 
სხვა ცვლადებისთვის: 
პირველი სტრატეგია: უბრალოდ საშუალო სახლის ფასი ავიღე training set-დან 
ამ მეთოდმა საკმაოდ ნორმალური მოდელი მოგვცა KFold validation-ით და rfe-ით, 0.024 mse validation-ზე საშუალოდ(rmse=0.16) და train-ის საშუალო 0.016 mse (rmse=0.13). თუმცა ამ სტრატეგიას აქვს თავისი ნაკლები, მაგალითად შესაძლო target leak
მეორე სტრატეგია: საშუალო ფასების smoothing
იმისთვის რომ data leak თავიდან აგვეცილებინა, გავაკეთე smoothing პირდაპირ საშუალოს მაგივრად ვხმარობდი შემდეგ ფორმულას: (n * mean + alpha * global_mean) / (n + alpha) სადაც alpha რაღაც კონსტანტა იყო n მოცემული კატეგორიული მნიშვნელობისთვის sample-ების რაოდენობა. შემდგომში alpha=20 და alpha=60 ვიყენებდი, თუმცა საბოლოოდ alpha=20-მა აჯობა. ეს მეთოდი დაგვაზღვევს იმისგან, რომ იშვიათი კატეგორიებისთვის target არ გაიleak-ოს
#NaN-ების შევსება 
აქ ვხმარობდით მარტივ სტრატეგიას: NaN-ებს ვავსებდით მედიანური მნიშვნელობებით. 
ცხადია ამ მონაცემების შევსებისას მხოლოდ train set-ს ვუყურებთ. იმ შემთხვევაში თუ ისეთი კატეგორია შეგვხვდება test-ში ან ვალიდაციაში რომელიც train-ში არ გვხვდებოდა, მას რიცხვითი სვეტებისთვის ისევ მედიანებით ვავსებთ, one-hot-encoding-ში კი უბრალოდ ყველა train-ში არსებული კატეგორიაში 0 გვაქვს.
